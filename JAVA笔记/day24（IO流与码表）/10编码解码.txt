字符串-->字节数组：编码
字节数组-->字符串：解码

如果便错了解不出来
如果编对了，解错了，有可能有救（gbk编对了，iso8解错了，有救）、（gbk编对了，utf-8解错了，一般没救）

你好：GBK:	-60 -29 -70 -60
你好：utf-8：	-28 -67 -96 -27 -91 -67

String str = "你好";
//编码：
byte[] buf = str.getBytes("utf-8");	//可以指定码表，一般有GBK和utf-8，默认是GBK
//解码：
String s1 = new String(buf,"GBK");	//如此乱码，因为utf-8三字节对应一中文，所以编出6字节
					//GBK两字节表示一中文（第一个字节第一位是1，第二个有正负），6字节会解出3中文
					//utf-8表示字符用最高3字节，就是表示英文跟中文所需字节数不同
					//utf-8编码的时候有编码头，编成的字节数不同个字节的编码头
					//不同，字符流解码的时候根据编码头来确定一次需要解多少个字节
