一、HashSet是一个实现了Set接口的类

二、HashSet集合数据结构：
该数据结构是哈希表，什么是哈希表，就是哈希算法算出来的所有结果的集合就是哈希表

该数据结构式如何分配存储位置的呢，就是先调用要分配位置的对象的哈希算法，算出位置，再判断该位置是否已有
对象，如没有，直接存放即可，如有，调用需要分配的对象的equals方法判断是否与该位置的对象相同，若相同直接不
存储（丢弃，也是为什么不重复），若不相同，执行衔接操作（在该位置开避一个分支，挨个存）。

所以，当要存储一个类的多个对象的时候，往往先要覆盖该类的hashCode和equals方法
其实，remove和contains底层调用的也是equals方法
Set类中调用判断是否相同要参考两方法：hashCode和equals
List只参考equals

三、LinkedHashSet类：
该类是HashSet的子类，该类的特点就是数据存储结构有序，即取出顺序和存入顺序是一致的
需要元素唯一又有序的时候，就用这个
为什么这个LinkedHashSet集合是有序的呢？
因为：
它的内部结构是哈希表与链表的结合，哈希表决定了存放的位置，而链表则决定着下一个元素是什么，就是前一个结点
存储着下一个结点的位置，先访问到前一个结点，才能访问下一个结点，因此，就能通过链表结构实现了有序化
